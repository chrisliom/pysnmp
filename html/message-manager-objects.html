Objects of manager class

Instances of manager class have the following methods:

encode_request(type, encoded_oids, encoded_vals)
decode_response(manager[, type])

The encode_request method takes the lists of BER encoded Object IDs encoded_oids and their associated values encoded_vals and builds complete SNMP message of specified type by applying generic.encode_bindings, generic.encode_snmp_pdu and generic.encode_message_sequence methods to the input data.

The following type argument values are recognized:

  • 'GETREQUEST' -- SNMP 'GET' request
  • 'GETNEXTREQUEST' -- SNMP 'GETNEXT' request
  • 'GETRESPONSE' -- SNMP 'GET' response
  • 'SETREQUEST' -- SNMP 'SET' request
  • 'TRAPREQUEST' -- SNMP 'TRAP' request

The encode_request method returns complete SNMP message as a string.

The decode_response method takes complete SNMP request (as produced by encode_request method) parses out and returns a tuple of the lists of BER encoded Object IDs encoded_oids and their associated values encoded_vals by applying the generic.decode_message_sequence, generic.decode_snmp_pdu and generic.decode_bindings methods to the input data.

Returned Object IDs and their respective values are matched against each other by their positions in the encoded_oids and encoded_vals lists.

The type argument may be used for verifying that the type of SNMP response matches type, or BadPDUType exception will be flagged.

The default value for type argument is 'GETRESPONSE'.

Objects of manager class have no public instance variables.
