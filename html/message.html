BER encoders & decoders for SNMP message

The pysnmp.message module defines manager, agent and trap classes, that implement BER encoders and decoders suitable for building and parsing SNMP messages specific to corresponding SNMP role.

These three classes can be safely subclassed in any combination whenever user application plays several roles at once. For example, a SNMP manager that also expects receiving traps might subclass the manager and trap classes.

The manager, agent and trap classes all rely on the generic class also defined in pysnmp.message module. This generic class implements BER encoders & decoders for common SNMP message components such as PDU and variable bindings. It, in turn, subclasses the ber.ber class (holding a set of BER encoders & decoders for SNMP subset of ASN.1 data types) so public attributes of ber.ber class become available to a user of these classes defined in pysnmp.message module.

The pysnmp.message module defines the following items:

manager([community[, version]])

Returns a new instance of manager class, representing a BER engine capable to build SNMP request and parse SNMP response messages.

The default value for SNMP community name community is "public".

The default for SNMP version is 0 (what indicates SNMP version 1).

agent([community[, version]])

Returns a new instance of agent class, representing a BER engine capable to parse SNMP request and build SNMP response messages.

The default value for SNMP community name community is "public".

The default for SNMP version is 0 (what indicates SNMP version 1).

trap([community[, version]])

Returns a new instance of trap class, representing a BER engine capable to build SNMP trap request and parse SNMP trap response messages.

The default value for SNMP community name community is "public".

The default for SNMP version is 0 (what indicates SNMP version 1).

generic()

Returns a new instance of generic class, representing a BER engine capable to build and parse SNMP message components (such as PDU, variables bindings).
