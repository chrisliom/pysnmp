Basic SNMP engine

The pysnmp.session module defines manager and agent classes that implement SNMP engines capable to send and receive (manager) and listen for (agent) SNMP messages over a TCP/IP network.

These two classes can be safely subclassed in any combination whenever user application plays both roles at once. For example, a SNMP proxy server, consisting of SNMP manager and agent, would subclass both manager and agent classes to achieve proxy functionality.

To add SNMP message processing capabilities to their own transport facilities, the manager and agent classes subclass message.manager and message.agent classes respectively. Also, both classes rely on message.trap class to handle SNMP traps.

The pysnmp.session module defines the following items:

manager(agent[, community])

Returns a new instance of manager class, representing SNMP manager session with SNMP agent process running on host agent.

The agent argument is a tuple representing connection endpoint in 'socket' module fashion -- ('hostname', port) where hostname a string and port is an integer.

The default value for SNMP community name community is "public".

An instance of the manager class can be reused for performing a number of SNMP requests to the same SNMP party.

agent([interface[, community]])

Returns a new instance of the agent class, representing SNMP agent listening for SNMP messages on local interface to come from network.

The default for the interface is to listen on loopback interface so the default value is ('127.0.0.1', 161).

Be aware that for a UNIX process to bind(2) on port 161 (less than 1024) superuser privelege is required. That's why it's strongly suggested to use a non-priveleged port whenever possible.

The default value for SNMP community name community is "public".

An instance of the agent class can be reused for serving many SNMP requests coming from different SNMP parties.
